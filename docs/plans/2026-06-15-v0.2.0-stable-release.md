# RouteSmith v0.2.0 Stable Release — Master Plan

> **SKILL:** Use subagent-driven-development to execute each section.
> **BRANCHING:** feature/* → dev → uat → dev-merge (CI/CD gates between each)
> **RESUMABLE:** Plan tracker + commits track progress. Worktree at `.worktrees/v0.2.0-stable`

**Goal:** Ship a stable v0.2.0 with core resilience, production hardening, live testing, and documentation.

**Architecture:** Each section is a self-contained plan dispatched to a subagent. Sections run sequentially (each builds on prior). Within each section, tasks are ordered by dependency.

**Tech Stack:** Python 3.13, LiteLLM, pytest, Prometheus, Docker, GitHub Actions, mkdocs

**Branch Strategy:**
```
main ──────────────────────────────────────────────
dev  ──●──●──●──●──●──●──●──●──●──●──●──●──●──●──
       \    \    \    \    \    \
   feature/a  feature/b  ...  (feature branches)
       ↓    ↓    ↓
   CI gate  CI gate  (must pass before merge)
         ↓      ↓
       dev   dev
         \     \
          uat-a  uat-b  (UAT smoke test)
                ↓
              merge to dev
```

## Quality Gates Between Sections

| Gate | Trigger | Checks |
|------|---------|--------|
| **CI gate** | PR from feature → dev | ruff, mypy, pytest 544 tests, bandit, pip-audit |
| **UAT gate** | PR from dev → uat branch | Live smoke test with Groq, manual review |
| **Merge gate** | PR from uat → dev | All above + code review |
