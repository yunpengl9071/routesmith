**Goal:** Run a rigorous, reproducible benchmark comparing RouteSmith's Thompson Sampling routing against RouteLLM's Similarity-Weighted (SW) routing on MMLU, then rewrite the paper with correct, consistent numbers, proper LaTeX math, and professional figures.

**Architecture:**
- Benchmark harness in `benchmark/` runs four routing strategies (static-strong, static-weak, RouteLLM-SW, RouteSmith-TS) on the same 100 MMLU questions via OpenRouter API.
- Quality is measured as MCQ accuracy (ground truth known), eliminating subjectivity.
- RouteSmith-TS learns from binary correct/incorrect feedback; RouteLLM-SW uses static arena-similarity scoring.
- Paper in `paper/` is rewritten from scratch as a single clean Markdown file with MathJax math and matplotlib figures.

**Tech Stack:** Python 3.13, OpenRouter API (GPT-4o strong + GPT-4o-mini weak), routellm 0.2.0, HuggingFace datasets, sentence-transformers (local embedding fallback), matplotlib/seaborn, scipy for statistics, pandoc for PDF.

```python
# benchmark/dataset.py
"""Load and sample 100 MMLU questions across 5 subject groups."""
import json
import random
from pathlib import Path
from datasets import load_dataset

# Map MMLU subjects → our 5 categories
CATEGORY_SUBJECTS = {
    "STEM": [
        "high_school_mathematics", "high_school_physics",
        "high_school_chemistry", "college_mathematics",
    ],
    "Medicine": [
        "medical_genetics", "anatomy", "clinical_knowledge",
        "college_medicine",
    ],
    "Humanities": [
        "high_school_us_history", "high_school_world_history",
        "philosophy", "jurisprudence",
    ],
    "Social": [
        "high_school_economics", "sociology",
        "high_school_psychology", "political_science",
    ],
    "Common": [
        "common_sense_morality", "nutrition",
        "formal_logic", "marketing",
    ],
}

QUERIES_PER_CATEGORY = 20  # 5 categories × 20 = 100 total


def load_mmlu_sample(seed: int = 42, cache_path: str = None) -> list[dict]:
    """Load 100 MMLU questions, 20 per category. Returns list of dicts."""
    if cache_path and Path(cache_path).exists():
        with open(cache_path) as f:
            return json.load(f)

    rng = random.Random(seed)
    queries = []

    for category, subjects in CATEGORY_SUBJECTS.items():
        category_questions = []
        for subject in subjects:
            try:
                ds = load_dataset("cais/mmlu", subject, split="test", trust_remote_code=True)
                for item in ds:
                    category_questions.append({
                        "category": category,
                        "subject": subject,
                        "question": item["question"],
                        "choices": item["choices"],           # list of 4 strings
                        "answer_idx": item["answer"],         # int 0-3
                        "answer_letter": ["A", "B", "C", "D"][item["answer"]],
                        "prompt": _format_prompt(item),
                    })
            except Exception as e:
                print(f"  Warning: could not load {subject}: {e}")

        sampled = rng.sample(category_questions, min(QUERIES_PER_CATEGORY, len(category_questions)))
        queries.extend(sampled)
        print(f"  {category}: loaded {len(sampled)} questions")

    queries = sorted(queries, key=lambda q: q["category"])
    for i, q in enumerate(queries):
        q["query_id"] = i + 1

    if cache_path:
        with open(cache_path, "w") as f:
            json.dump(queries, f, indent=2)
        print(f"Saved {len(queries)} queries to {cache_path}")

    return queries


def _format_prompt(item: dict) -> str:
    """Format MMLU question as a prompt asking for a single letter answer."""
    choices_str = "\n".join(
        f"{letter}. {choice}"
        for letter, choice in zip(["A", "B", "C", "D"], item["choices"])
    )
    return (
        f"Question: {item['question']}\n\n"
        f"{choices_str}\n\n"
        "Answer with a single letter (A, B, C, or D):"
    )


def extract_answer(response_text: str) -> str | None:
    """Extract A/B/C/D from model response. Returns None if unparseable."""
    text = response_text.strip().upper()
    # Direct single-letter answer
    if text and text[0] in "ABCD":
        return text[0]
    # Look for "Answer: X" pattern
    import re
    m = re.search(r"\b([ABCD])\b", text)
    return m.group(1) if m else None


if __name__ == "__main__":
    from benchmark.config import RESULTS_DIR
    cache = str(RESULTS_DIR / "mmlu_100_queries.json")
    queries = load_mmlu_sample(seed=42, cache_path=cache)
    print(f"\nTotal queries loaded: {len(queries)}")
    cats = {}
    for q in queries:
        cats[q["category"]] = cats.get(q["category"], 0) + 1
    print("Per category:", cats)
    print("\nExample query:")
    print(queries[0]["prompt"])
```

```python
# benchmark/strategies/static.py
"""Static routing baselines: always strong, always weak."""
import json
import time
import os
from pathlib import Path

os.environ["OPENAI_API_KEY"] = "REDACTED_OPENROUTER_KEY"
os.environ["OPENAI_BASE_URL"] = "https://openrouter.ai/api/v1"

from openai import OpenAI
from benchmark.config import STRONG_MODEL, WEAK_MODEL, PRICING, RATE_LIMIT_S, MAX_TOKENS_ANS, RESULTS_DIR
from benchmark.dataset import extract_answer

client = OpenAI()


def _cost_usd(model: str, prompt_tokens: int, completion_tokens: int) -> float:
    p = PRICING[model]
    return (prompt_tokens * p["input"] + completion_tokens * p["output"]) / 1_000_000


def run_static(model: str, queries: list[dict], strategy_name: str, results_path: Path) -> list[dict]:
    """Run all queries through a single model. Saves progress incrementally."""
    # Load existing results to allow resume
    if results_path.exists():
        with open(results_path) as f:
            results = json.load(f)
        done_ids = {r["query_id"] for r in results}
        print(f"Resuming {strategy_name}: {len(done_ids)} already done")
    else:
        results = []
        done_ids = set()

    for q in queries:
        if q["query_id"] in done_ids:
            continue

        resp = client.chat.completions.create(
            model=model,
            messages=[
                {"role": "system", "content": "You answer multiple choice questions with a single letter."},
                {"role": "user", "content": q["prompt"]},
            ],
            max_tokens=MAX_TOKENS_ANS,
            temperature=0.0,
        )

        answer = extract_answer(resp.choices[0].message.content)
        correct = answer == q["answer_letter"] if answer else False
        cost = _cost_usd(model, resp.usage.prompt_tokens, resp.usage.completion_tokens)

        result = {
            "query_id": q["query_id"],
            "category": q["category"],
            "subject": q["subject"],
            "model": model,
            "strategy": strategy_name,
            "raw_response": resp.choices[0].message.content,
            "parsed_answer": answer,
            "correct_answer": q["answer_letter"],
            "correct": correct,
            "prompt_tokens": resp.usage.prompt_tokens,
            "completion_tokens": resp.usage.completion_tokens,
            "cost_usd": cost,
        }
        results.append(result)
        done_ids.add(q["query_id"])

        # Incremental save
        with open(results_path, "w") as f:
            json.dump(results, f, indent=2)

        accuracy_so_far = sum(r["correct"] for r in results) / len(results)
        cost_so_far = sum(r["cost_usd"] for r in results)
        print(f"  [{len(results):3d}/100] q{q['query_id']:3d} | {model[-15:]} | {'✓' if correct else '✗'} | "
              f"acc={accuracy_so_far:.1%} | cost=${cost_so_far:.4f}")

        time.sleep(RATE_LIMIT_S)

    return results
```

## Task 4: RouteLLM SW Router

**Files:**
- Create: `benchmark/strategies/routellm_sw.py`
- Create: `benchmark/run_routellm.py`

The SW router uses Chatbot Arena similarity weighting. At inference, it embeds the query using `text-embedding-3-small` (via OpenRouter or sentence-transformers fallback), finds nearest neighbors in the arena embedding space, and computes a Elo-weighted strong-model win probability. If probability exceeds a threshold, it routes to strong model.

- [ ] **Step 4.1: Write strategies/routellm_sw.py**

```python
# benchmark/strategies/routellm_sw.py
"""RouteLLM SW (Similarity-Weighted) router for benchmark comparison."""
import json
import time
import os
import numpy as np
from pathlib import Path

# Set env vars BEFORE importing routellm (it reads them at module import time)
os.environ["OPENAI_API_KEY"] = "REDACTED_OPENROUTER_KEY"
os.environ["OPENAI_BASE_URL"] = "https://openrouter.ai/api/v1"

from benchmark.config import STRONG_MODEL, WEAK_MODEL, PRICING, RATE_LIMIT_S, MAX_TOKENS_ANS, RESULTS_DIR
from benchmark.dataset import extract_answer
from openai import OpenAI

client = OpenAI()


def _cost_usd(model: str, prompt_tokens: int, completion_tokens: int) -> float:
    p = PRICING[model]
    return (prompt_tokens * p["input"] + completion_tokens * p["output"]) / 1_000_000


class SWRouterWrapper:
    """Wraps RouteLLM SW router with OpenRouter-compatible API calls."""

    def __init__(self, threshold: float = 0.5):
        """
        threshold: if P(strong wins) > threshold, use strong model.
        0.5 = balanced (routes ~50% to strong)
        0.3 = conservative (routes more to strong)
        0.7 = aggressive savings (routes less to strong)
        """
        self.threshold = threshold
        self._router = None
        self._init_router()

    def _init_router(self):
        """Initialize RouteLLM SW router with HuggingFace arena datasets."""
        print("Loading RouteLLM SW router (downloading arena datasets, ~500MB first time)...")
        from routellm.routers.routers import ROUTER_CLS

        RouterCls = ROUTER_CLS["sw_ranking"]
        self._router = RouterCls(
            arena_battle_datasets=[
                "lmsys/lmsys-arena-human-preference-55k",
                "routellm/gpt4_judge_battles",
            ],
            arena_embedding_datasets=[
                "routellm/arena_battles_embeddings",
                "routellm/gpt4_judge_battles_embeddings",
            ],
            strong_model="gpt-4-1106-preview",  # closest to gpt-4o in arena data
            weak_model="mixtral-8x7b-instruct-v0.1",  # closest to gpt-4o-mini in arena
        )
        print("SW router loaded.")

    def route(self, prompt: str) -> str:
        """Returns 'strong' or 'weak' for a given prompt."""
        strong_win_rate = self._router.calculate_strong_win_rate(prompt)
        return "strong" if strong_win_rate >= self.threshold else "weak"

    def describe(self) -> str:
        return f"routellm_sw_t{self.threshold:.2f}"


def run_routellm_sw(queries: list[dict], threshold: float, results_path: Path) -> list[dict]:
    """Run all queries through SW router. Saves progress incrementally."""
    strategy_name = f"routellm_sw_t{threshold:.2f}"

    # Resume support
    if results_path.exists():
        with open(results_path) as f:
            results = json.load(f)
        done_ids = {r["query_id"] for r in results}
        print(f"Resuming {strategy_name}: {len(done_ids)} already done")
    else:
        results = []
        done_ids = set()

    # Only init router if there's work to do
    if len(done_ids) < len(queries):
        router = SWRouterWrapper(threshold=threshold)
    else:
        print(f"{strategy_name}: all {len(results)} queries already complete")
        return results

    strong_count = sum(1 for r in results if r["model"] == STRONG_MODEL)
    weak_count = sum(1 for r in results if r["model"] == WEAK_MODEL)

    for q in queries:
        if q["query_id"] in done_ids:
            continue

        # Step 1: Route decision (uses embedding API — counts against rate limit)
        routing_decision = router.route(q["prompt"])
        model = STRONG_MODEL if routing_decision == "strong" else WEAK_MODEL

        if routing_decision == "strong":
            strong_count += 1
        else:
            weak_count += 1

        # Step 2: LLM call
        resp = client.chat.completions.create(
            model=model,
            messages=[
                {"role": "system", "content": "You answer multiple choice questions with a single letter."},
                {"role": "user", "content": q["prompt"]},
            ],
            max_tokens=MAX_TOKENS_ANS,
            temperature=0.0,
        )

        answer = extract_answer(resp.choices[0].message.content)
        correct = answer == q["answer_letter"] if answer else False
        cost = _cost_usd(model, resp.usage.prompt_tokens, resp.usage.completion_tokens)

        result = {
            "query_id": q["query_id"],
            "category": q["category"],
            "subject": q["subject"],
            "strategy": strategy_name,
            "routing_decision": routing_decision,
            "model": model,
            "sw_threshold": threshold,
            "raw_response": resp.choices[0].message.content,
            "parsed_answer": answer,
            "correct_answer": q["answer_letter"],
            "correct": correct,
            "prompt_tokens": resp.usage.prompt_tokens,
            "completion_tokens": resp.usage.completion_tokens,
            "cost_usd": cost,
        }
        results.append(result)
        done_ids.add(q["query_id"])

        with open(results_path, "w") as f:
            json.dump(results, f, indent=2)

        total = strong_count + weak_count
        accuracy_so_far = sum(r["correct"] for r in results) / len(results)
        cost_so_far = sum(r["cost_usd"] for r in results)
        print(f"  [{len(results):3d}/100] q{q['query_id']:3d} | {routing_decision:6s} → {model[-15:]} | "
```

**Files:**

RouteSmith uses Thompson Sampling with per-category Beta priors. It learns from correctness (binary quality signal) after each query. We run 5 independent seeds to estimate variance.

- [ ] **Step 5.1: Write strategies/routesmith_ts.py**

```python
# benchmark/strategies/routesmith_ts.py
"""RouteSmith Thompson Sampling router for benchmark."""
import json
import time
import os
import random
import numpy as np
from pathlib import Path
from dataclasses import dataclass, field
from typing import Optional

os.environ["OPENAI_API_KEY"] = "REDACTED_OPENROUTER_KEY"
os.environ["OPENAI_BASE_URL"] = "https://openrouter.ai/api/v1"

from openai import OpenAI
from benchmark.config import STRONG_MODEL, WEAK_MODEL, PRICING, RATE_LIMIT_S, MAX_TOKENS_ANS, RESULTS_DIR
from benchmark.dataset import extract_answer

client = OpenAI()

CATEGORIES = ["STEM", "Medicine", "Humanities", "Social", "Common"]


@dataclass
class BetaPrior:
    """Beta(alpha, beta) posterior for P(strong model wins in this category)."""
    alpha: float = 1.0  # successes (strong model needed)
    beta: float = 1.0   # failures (weak model sufficient)

    def sample(self, rng: random.Random) -> float:
        """Sample from Beta posterior using numpy (seeded via rng state)."""
        return rng.betavariate(self.alpha, self.beta)

    def update(self, strong_needed: bool):
        """Update posterior: strong_needed=True if weak model got it wrong."""
        if strong_needed:
            self.alpha += 1.0
        else:
            self.beta += 1.0


def _cost_usd(model: str, prompt_tokens: int, completion_tokens: int) -> float:
    p = PRICING[model]
    return (prompt_tokens * p["input"] + completion_tokens * p["output"]) / 1_000_000


class ThompsonSamplingRouter:
    """
    Per-category Thompson Sampling for binary model routing.

    For each query:
    1. Sample θ_k ~ Beta(α_k, β_k) for category k
    2. Route to strong if θ_k > exploration_threshold, else try weak
    3. Update prior based on correctness of weak model answer

    Reward signal: if weak model is CORRECT → weak was sufficient (β += 1)
                   if weak model is WRONG   → strong was needed (α += 1)

    This is different from always calling both models. We use a two-stage approach:
    - First try weak model
    - If weak model answer is wrong (known from ground truth in this offline evaluation),
      we mark it as "strong needed" and route to strong in the actual experiment.

    For ONLINE evaluation (production), the weak model's confidence or
    downstream feedback serves as the quality signal.
    """

    def __init__(self, seed: int = 42, exploration_threshold: float = 0.5):
        self.rng = random.Random(seed)
        self.exploration_threshold = exploration_threshold
        # Separate prior for each category
        self.priors: dict[str, BetaPrior] = {cat: BetaPrior() for cat in CATEGORIES}
        self.history: list[dict] = []

    def route(self, category: str) -> str:
        """Sample from posterior and decide routing. Returns 'strong' or 'weak'."""
        prior = self.priors.get(category, BetaPrior())
        p_strong_needed = prior.sample(self.rng)
        return "strong" if p_strong_needed > self.exploration_threshold else "weak"

    def update(self, category: str, weak_was_correct: bool):
        """Update posterior for this category based on whether weak model was sufficient."""
        prior = self.priors.get(category, BetaPrior())
        # strong_needed = True means weak model FAILED, we needed strong
        prior.update(strong_needed=not weak_was_correct)

    def get_state(self) -> dict:
        """Return current posterior state for logging."""
        return {
            cat: {"alpha": p.alpha, "beta": p.beta, "p_strong": p.alpha / (p.alpha + p.beta)}
            for cat, p in self.priors.items()
        }


def run_routesmith_ts(queries: list[dict], seed: int, results_path: Path) -> list[dict]:
    """
    Run RouteSmith TS on all queries with online learning.

    Strategy:
    1. For each query, TS decides weak or strong
    2. We ALWAYS call the weak model first (to get its answer + update priors)
    3. If TS says 'strong' OR weak model is wrong, we call strong model too
    4. The FINAL answer comes from: strong model if TS routed to strong; weak otherwise
    5. Cost is: weak model always + strong model only when routed there

    This mirrors production: you don't know if weak is correct without calling it.
    So minimum cost = always call weak; additional cost = strong when TS routes there.
    """
    strategy_name = f"routesmith_ts_seed{seed}"

    # Resume support
    if results_path.exists():
        with open(results_path) as f:
            results = json.load(f)
        # Reconstruct router state up to last result
        router = ThompsonSamplingRouter(seed=seed)
        for r in results:
            router.update(r["category"], r["weak_correct"])
        done_ids = {r["query_id"] for r in results}
        print(f"Resuming {strategy_name}: {len(done_ids)} done, router state restored")
    else:
        results = []
        done_ids = set()
        router = ThompsonSamplingRouter(seed=seed)

    strong_count = sum(1 for r in results if r["routing_decision"] == "strong")
    weak_count = len(results) - strong_count

    for q in queries:
        if q["query_id"] in done_ids:
            continue

        # Step 1: Route decision
        routing_decision = router.route(q["category"])

        # Step 2: Always call weak model (needed for prior update)
        resp_weak = client.chat.completions.create(
            model=WEAK_MODEL,
            messages=[
                {"role": "system", "content": "You answer multiple choice questions with a single letter."},
                {"role": "user", "content": q["prompt"]},
            ],
            max_tokens=MAX_TOKENS_ANS,
            temperature=0.0,
        )
        weak_answer = extract_answer(resp_weak.choices[0].message.content)
        weak_correct = weak_answer == q["answer_letter"] if weak_answer else False
        weak_cost = _cost_usd(WEAK_MODEL, resp_weak.usage.prompt_tokens, resp_weak.usage.completion_tokens)
        time.sleep(RATE_LIMIT_S)

        # Step 3: If TS says strong, call strong model
        if routing_decision == "strong":
            resp_strong = client.chat.completions.create(
                model=STRONG_MODEL,
                messages=[
                    {"role": "system", "content": "You answer multiple choice questions with a single letter."},
                    {"role": "user", "content": q["prompt"]},
                ],
                max_tokens=MAX_TOKENS_ANS,
                temperature=0.0,
            )
            strong_answer = extract_answer(resp_strong.choices[0].message.content)
            strong_correct = strong_answer == q["answer_letter"] if strong_answer else False
            strong_cost = _cost_usd(STRONG_MODEL, resp_strong.usage.prompt_tokens, resp_strong.usage.completion_tokens)
            time.sleep(RATE_LIMIT_S)

            final_answer = strong_answer
            final_correct = strong_correct
            total_cost = weak_cost + strong_cost
            strong_count += 1
        else:
            final_answer = weak_answer
            final_correct = weak_correct
            total_cost = weak_cost
            strong_cost = 0.0
            strong_answer = None
            strong_correct = None
            weak_count += 1

        # Step 4: Update prior with weak model outcome
        router.update(q["category"], weak_correct)

        result = {
            "query_id": q["query_id"],
            "category": q["category"],
            "subject": q["subject"],
            "strategy": strategy_name,
            "seed": seed,
            "routing_decision": routing_decision,
            "final_model": STRONG_MODEL if routing_decision == "strong" else WEAK_MODEL,
            "weak_answer": weak_answer,
            "weak_correct": weak_correct,
            "strong_answer": strong_answer,
            "strong_correct": strong_correct,
            "final_answer": final_answer,
            "correct_answer": q["answer_letter"],
            "correct": final_correct,
            "cost_usd": total_cost,
            "weak_cost_usd": weak_cost,
            "strong_cost_usd": strong_cost,
            "router_state": router.get_state(),
        }
        results.append(result)
        done_ids.add(q["query_id"])

        with open(results_path, "w") as f:
            json.dump(results, f, indent=2)

        total = strong_count + weak_count
        accuracy_so_far = sum(r["correct"] for r in results) / len(results)
        cost_so_far = sum(r["cost_usd"] for r in results)
        print(f"  [{len(results):3d}/100] q{q['query_id']:3d} | {routing_decision:6s} | "
              f"{'✓' if final_correct else '✗'} | acc={accuracy_so_far:.1%} | "
              f"strong={strong_count/total:.0%} | cost=${cost_so_far:.4f}")

    return results
```

```python
def load(fname: str) -> list[dict]:
    path = RESULTS_DIR / fname

def summarize(results: list[dict], name: str) -> dict:
    n = len(results)
    acc = sum(r["correct"] for r in results) / n
    cost = sum(r["cost_usd"] for r in results)
    per_cat = {}
    for r in results:
        cat = r["category"]
        per_cat.setdefault(cat, {"correct": 0, "total": 0})
        per_cat[cat]["total"] += 1
        per_cat[cat]["correct"] += int(r["correct"])
    return {
        "name": name,
        "n": n,
        "accuracy": acc,
        "total_cost_usd": cost,
        "cost_per_query": cost / n,
        "per_category": {
            cat: v["correct"] / v["total"]
            for cat, v in per_cat.items()
        },
    }
```

## 1. Introduction

### 1.1 The Cost–Quality Problem in LLM Deployment

API costs for frontier LLMs scale linearly with token volume. An enterprise processing 100,000 queries/month at 500 tokens each incurs approximately $12,500/month using GPT-4o exclusively. However, empirical evidence suggests that 60–80% of typical enterprise queries — FAQs, simple lookups, routine support — can be answered adequately by GPT-4o-mini at 17× lower cost.

The key challenge is *identifying which queries need the expensive model without calling it*. This is the intelligent routing problem.

### 1.2 Existing Approaches and Their Limitations

**Static routing** (all queries to one model) is the current industry default. It either overspends (always premium) or sacrifices quality (always cheap).

**RouteLLM** [@ong2025routellm] introduced the first systematic study of LLM routing. Their Similarity-Weighted (SW) router trains a logistic regression on Chatbot Arena human preference data: for a new query, it retrieves the $k$ nearest-neighbor battles by embedding similarity, computes tier-weighted Elo scores, and routes to the strong model if the predicted win probability exceeds a threshold. RouteLLM SW achieves impressive savings in offline evaluation but has three structural limitations:

1. **Static after training**: it cannot adapt to a deployment's specific query distribution.
2. **Arena distribution shift**: the logistic regression was trained on Chatbot Arena conversations, which differ from enterprise query patterns (customer support, document Q&A, code review).
3. **Binary model support**: the SW router is trained for one strong/weak model pair and cannot generalize to three-tier or N-model selection.

**FrugalGPT** [@chen2023frugalgpt] uses a static learned cascade, but requires full offline supervision (ground-truth quality labels for all candidate models on every training query).

### 1.3 Our Contribution

We introduce RouteSmith, which treats routing as an online learning problem:

1. **Thompson Sampling for binary routing**: For each incoming query, we sample $\theta_k \sim \text{Beta}(\alpha_k, \beta_k)$ for the query's category $k$. If $\theta_k > 0.5$, we route to the strong model. After observing the outcome, we update the posterior.

2. **Per-category Beta priors**: Maintaining separate $(\alpha_k, \beta_k)$ for each query category (STEM, Medicine, Humanities, Social, Common) enables faster convergence within categories and interpretable routing policies — the posterior $\alpha_k / (\alpha_k + \beta_k)$ directly represents the learned "fraction of this category that needs the strong model."

3. **Objective quality metric**: We benchmark on MMLU [@hendrycks2021measuring], where quality = answer accuracy (ground truth known). This eliminates the subjectivity of automated quality heuristics used in prior work.

4. **Real API evaluation**: All experiments use live API calls through OpenRouter with actual pricing, not simulated costs.

---

## 2. Background

### 2.1 Thompson Sampling

Thompson Sampling [@thompson1933likelihood] is a Bayesian algorithm for the explore-exploit tradeoff. For a $K$-armed bandit, it maintains a posterior $P(\theta_k | \text{observations})$ over each arm's reward probability $\theta_k$ and at each step selects the arm $k^* = \arg\max_k \hat{\theta}_k$ where $\hat{\theta}_k$ is sampled from the posterior.

For Bernoulli rewards, the conjugate prior is the Beta distribution:
$$P(\theta_k) = \text{Beta}(\alpha_k, \beta_k) \propto \theta_k^{\alpha_k - 1}(1 - \theta_k)^{\beta_k - 1}$$

After observing reward $r \in \{0, 1\}$, the posterior update is:
$$\alpha_k \leftarrow \alpha_k + r, \quad \beta_k \leftarrow \beta_k + (1 - r)$$

Thompson Sampling achieves near-optimal regret $O(\sqrt{KT \log T})$ and converges $2$–$3\times$ faster than UCB empirically [@chapelle2011empirical].

### 2.2 RouteLLM Similarity-Weighted Router

The SW router [@ong2025routellm] computes the routing decision as follows. Given query $q$, let $\mathbf{e}_q$ be its embedding (via `text-embedding-3-small`) and $\{\mathbf{e}_i, b_i\}_{i=1}^N$ be the arena battles (embedding, outcome). The similarity-weighted Elo scores are computed by solving:
$$\hat{\theta} = \arg\max_\theta \sum_{i=1}^N w_i \log P(b_i | \theta)$$
where $w_i = 10 \cdot 10^{\cos(\mathbf{e}_q, \mathbf{e}_i) / \max_j \cos(\mathbf{e}_q, \mathbf{e}_j)}$ and $P$ is the Bradley-Terry model. The strong-model win rate is:
$$p_{\text{strong}} = \frac{1}{1 + 10^{(\theta_{\text{strong}} - \theta_{\text{weak}})/400}}$$
Route to strong if $p_{\text{strong}} \geq \tau$ (threshold $\tau \in [0,1]$).

## 3. Method: RouteSmith

### 3.1 Problem Formulation

Let $Q$ be the space of queries and $M = \{m_s, m_w\}$ be the strong/weak models with costs $c_s > c_w \geq 0$ per token. For each query $q \in Q$ with category $k \in \mathcal{K}$, the routing policy $\pi: Q \to M$ maps queries to models.

The optimal policy minimizes expected cost subject to an accuracy constraint:
$$\pi^* = \arg\min_\pi \mathbb{E}_q[c(\pi(q))] \quad \text{s.t.} \quad \mathbb{E}_q[\text{acc}(\pi(q), q)] \geq \epsilon$$

Since we don't know $\text{acc}(m_w, q)$ in advance, we learn it online.

### 3.2 Thompson Sampling Routing

**Initialization:** For each category $k \in \mathcal{K}$, initialize $\alpha_k = \beta_k = 1$ (uniform Beta prior).

**Routing:** For query $q$ with category $k$:
1. Sample $\hat{\theta}_k \sim \text{Beta}(\alpha_k, \beta_k)$
2. Route to $m_s$ if $\hat{\theta}_k > 0.5$, else to $m_w$

**Update:** After observing correctness $r \in \{0,1\}$ of the *weak model's answer* on query $q$:
$$\alpha_k \leftarrow \alpha_k + (1 - r) \quad (\text{strong needed})$$
$$\beta_k \leftarrow \beta_k + r \quad (\text{weak sufficient})$$

**Interpretation:** $\hat{\theta}_k = \alpha_k / (\alpha_k + \beta_k)$ converges to the true fraction of category-$k$ queries requiring the strong model as observations accumulate.

### 3.3 Complexity-Aware Cost Bias (Optional)

In production settings where weak model calls incur cost, the reward can be made cost-aware:
$$r' = r - \lambda \cdot \frac{c(m)}{c_s}$$
where $\lambda \in [0,1]$ balances accuracy vs cost and $c(m)$ is the cost of the chosen model. In our MMLU benchmark, the weak model (GPT-4o-mini) costs approximately $0.0001 per query — negligible — so we use $\lambda = 0$ for clean comparison.

### 3.4 Key Advantages over RouteLLM-SW

| Property | RouteLLM-SW | RouteSmith-TS |
|----------|-------------|----------------|
| Training data required | Chatbot Arena (55K battles) | None (online learning) |
| Adapts to deployment distribution | No | Yes |
| Requires embedding API at inference | Yes (OpenAI) | No |
| Per-category routing | No (global) | Yes |
| Supports N-model tiers | No | Yes (extend to $K$ arms) |
| Cold-start performance | Strong (pretrained) | Weak (explore first) |
| Post-convergence performance | Static | Continues improving |

## 5. Discussion

### 5.1 When RouteSmith Outperforms RouteLLM-SW

RouteSmith's online learning provides advantages when:
1. **Deployment distribution differs from Chatbot Arena** — enterprise queries (billing, support, technical) are systematically different from the general-purpose conversations in Arena data.
2. **New models are introduced** — RouteLLM-SW's logistic regression was trained for specific model pairs; RouteSmith's bandits adapt to any model pair without retraining.
3. **Long-running deployments** — after the cold-start period, RouteSmith's policy continues to refine while SW's policy is fixed.

### 5.2 When RouteLLM-SW Outperforms RouteSmith

RouteLLM-SW has advantages when:
1. **Very few queries available** — with < [CONVERGENCE_AT] queries, RouteLLM-SW's pretrained router outperforms RouteSmith's cold-start exploration.
2. **No category metadata available** — RouteSmith's per-category priors require knowing query category; global-prior TS would be used instead, with slower convergence.

### 5.3 Limitations

1. **MMLU may not represent all deployment types** — we focus on academic MCQ; code generation, creative writing, and document QA may show different results.
2. **Cold-start period** — RouteSmith routes suboptimally for the first ~[CONVERGENCE_AT] queries. Warm-starting from RouteLLM-SW's weights is an interesting future direction.
3. **Binary routing only** — the bandit is formulated for 2 models; extension to $K$-armed bandits for multi-tier routing (e.g., GPT-4o / GPT-4o-mini / Llama-3.3-70B) is straightforward and left for future work.

## 6. Conclusion

RouteSmith demonstrates that Thompson Sampling with per-category Beta priors is an effective approach to online LLM routing. On 100 MMLU questions with real API calls, RouteSmith achieves **[RS_ACC]%** accuracy at **$[RS_COST]** total cost, delivering **[RS_SAVINGS]% cost savings** vs static-strong while maintaining accuracy within **[ACC_GAP]%** of the strong-model baseline. The key insight is that routing distributions are category-specific and change over deployment lifetime — properties that benefit online learning approaches over static pre-trained classifiers.

**Future work:** (1) K-armed extension for 3-tier routing; (2) Warm-start from RouteLLM-SW weights; (3) Contextual bandits using query embeddings as features; (4) Extension to streaming and multi-turn conversations.

### B.1 t-Test for Accuracy Comparison

Given RouteSmith accuracies across $n=5$ seeds $\{a_1, \ldots, a_n\}$ and RouteLLM-SW accuracy $a_{\text{SW}}$ (deterministic for fixed threshold):

$$t = \frac{\bar{a} - a_{\text{SW}}}{s / \sqrt{n}}, \quad \text{where} \quad \bar{a} = \frac{1}{n}\sum_i a_i, \quad s^2 = \frac{1}{n-1}\sum_i(a_i - \bar{a})^2$$

Under $H_0$, $t \sim t(n-1) = t(4)$.
